## 产品简介
### 一句话说明
Temu 商品发布自动化是一套“从选品表到批量发布”的全自动执行助手，覆盖搜索、认领、18 步批量编辑、店铺选择和发布确认，确保 5 分钟内得到可上架的 40 条商品素材。

### 工作方式（通俗视角）
```

根据选品表完成妙手采集之后
csv 文件 → 系统读取并生成任务
    ↓
    妙手首次编辑5个产品
    ↓
    妙手批量编辑18步
    ↓
    妙手发布40个产品
```

## 核心能力一览
| 模块 | 系统做什么 | 对运营的意义 |
| --- | --- | --- |
| 数据接入 | 读取标准选品表、自动补充缺失项 | 无需手动拆表或改格式 |
| 批量编辑 18 步 | 固定值（产地、包装等）、智能值（重量/尺寸）、AI 标题、PDF 说明书 | 一次点击涵盖全部合规字段 |
| 店铺/供货价 | 按倍率自动计算并同步 | 保证价格策略统一 |
| 结果回传 | 输出 JSON、日志、截图、统计报表 | 方便业务、技术、管理层复核 |

## 启用前准备清单
- ✅ **账号**：可用的 Temu 商家后台账号（含妙手采集箱权限），提前完成短信/邮箱验证。
- 按照选品表完成采集确保妙手采集箱内有5个产品
- 将选品表csv文件发给技术同事

## 标准工作流程（非技术视角）
1. **准备选品表**  
   - 复制模板，填入新品名称、成本、关键词、图片路径等。  
   - 确认每条商品有清晰图片、最少 1 个规格。

2. **整理素材**  
   - 将图片统一命名后压缩成压缩包
   - 将压缩包发给技术同事

3. **启动一次批量任务**  
   - 运营人员在终端执行预设命令（由技术同事或脚本双击触发）。  
   - 运行期间保持浏览器窗口前台，可随时看到系统自动点击。


4. **查看结果**  
   - 查看妙手采集箱内的产品数量是否为40个

## Web 管理面板（零指令）
> 适合电脑小白，一次配置即可长期复用。

1. **首次安装（一次性）**  
   - Windows：双击 `apps/temu-auto-publish/install_web_panel.bat`。  
   - macOS：双击 `apps/temu-auto-publish/install_web_panel.command`。  
   - 等待窗口提示 “DONE”，表示依赖已安装。

2. **每日运行**  
   - Windows：双击 `apps/temu-auto-publish/start_web_panel.bat`。  
   - macOS：双击 `apps/temu-auto-publish/start_web_panel.command`。  
   - 浏览器会自动打开 Web 管理页面，按界面提示完成 3 步（上传选品表 → 选择模式 → 点击“一键开始”）。

3. **过程提示**  
   - 页面实时显示“运行中 / 完成 / 失败”状态卡片，并滚动输出日志。  
   - “环境自检”按钮用于检查依赖、路径是否就绪。  
   - “示例选品表”链接可随时下载标准模板核对格式。  
   - 失败时复制整段红色日志给技术支撑即可，运营无需排查代码。

4. **注意事项**  
   - 单机一次仅允许跑一个任务，界面会提示“已有任务正在运行”。  
   - 上传的文件自动保存到 `apps/temu-auto-publish/data/input/web_panel/`，可直接复用。  
   - 默认浏览器隐藏运行，如需观察流程可在页面中将“浏览器模式”切换为“显示窗口”。
   - 如需 Windows “下载即用”版本，可在 Windows 环境执行 `uv run python apps/temu-auto-publish/build_windows_exe.py build` 生成 `TemuWebPanel.exe`，分发后双击即可启动。

## 风险与应对
- **账号登录失败**：检查账号状态或手动完成验证码；系统支持 Cookie 复用，必要时重新登录一次即可。
- **页面更新导致卡住**：根据提示截图交给技术同事调整选择器，通常当日可修复。
- **素材缺失**：系统会标记异常并跳过，该商品不会发布；补齐素材后重新执行。
- **说明书不合规**：建议使用项目提供的多语种模板或法务审阅版本，确保平台审核通过。

## 常见问题（FAQ）
**Q1：我不会写代码，如何触发流程？**  
A：技术同事会提供一键脚本或桌面快捷方式，你只需双击并保持浏览器前台即可。

**Q2：如何更换产品说明书文件？**  
A：将新的 PDF 放到 `apps/temu-auto-publish/data/manual/`，并在配置中指向该文件；未做配置时，系统默认选择最新修改的 PDF。

**Q3：批量任务失败后可以接着跑吗？**  
A：可以。系统会记录已成功的商品，再次执行时会自动跳过已完成项，避免重复认领。

**Q4：发布成功了，如何对外汇报？**  
A：将 `workflow_*.json` 通过内置脚本转换为 Excel（脚本附带在 `scripts/reporting/`，技术同事可代为导出），即可获得包含商品标题、店铺、供货价、发布时间的汇总表。
